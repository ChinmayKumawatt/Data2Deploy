DATA INGESTION FLOW

1. Start `initiate_data_ingestion()`.
2. Log that the ingestion stage has started.
3. Read `config.dataset_path`.
4. Check that the dataset path is not empty.
5. Check that the dataset path exists.
6. Check that the dataset path points to a file, not a directory.
7. Load the CSV into a pandas DataFrame.
8. Log the source path and original dataset size.
9. Check that the DataFrame is not empty.
10. Check that the DataFrame has columns.
11. Check that `config.target_column` exists in the dataset.
12. Drop duplicate rows.
13. Log dataset size before and after deduplication.
14. Re-check that the deduplicated DataFrame is still not empty.
15. Check that dataset row count is above the configured minimum.
16. Validate the target column values:
17. Ensure the target series is not empty.
18. Ensure the target series is not all null.
19. Ensure the target series does not contain any null values.
20. Validate split configuration:
21. Ensure `test_size` is a valid float or int.
22. If float, ensure it is between 0 and 1.
23. If int, ensure it is greater than 0 and smaller than dataset size.
24. Resolve the raw dataset save path.
25. Create the parent directory for the raw file if needed.
26. Save the full deduplicated dataset as `raw.csv`.
27. Split features and target:
28. `X = df.drop(target_column)`
29. `y = df[target_column]`
30. Decide whether stratified split can be used.
31. Skip stratify if task type is not classification.
32. Skip stratify if stratify is disabled in config.
33. Skip stratify if target has fewer than 2 unique values.
34. Skip stratify if any class has fewer than 2 samples.
35. Run `train_test_split()`.
36. Rebuild `train_df` and `test_df` by concatenating features and target.
37. Log the train and test shapes.
38. Create parent directories for `train_path` and `test_path` if needed.
39. Save `train_df` to `config.train_path`.
40. Save `test_df` to `config.test_path`.
41. Log that train and test files were saved successfully.
42. Return `DataIngestionArtifact`.
43. Artifact includes:
44. `train_file_path`
45. `test_file_path`
46. `raw_file_path`
47. If any step fails, wrap the error in `CustomException`.

DEBUGGING CHECKPOINTS

1. Confirm `config.dataset_path` is correct.
2. Confirm the CSV actually contains the target column.
3. Check whether deduplication removes too many rows.
4. Check whether the minimum row constraint is too strict.
5. Check whether nulls in the target column stop ingestion.
6. Check whether `test_size` is valid for the current dataset size.
7. Check whether stratification is skipped because a class has too few samples.
8. Confirm the output files were written to the expected locations.
